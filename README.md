# ✦ Suluu (Suluu)

**Маркетплейс beauty-услуг в Кыргызстане**

Единая платформа для салонов красоты, мастеров и клиентов. Позволяет записываться на услуги, управлять расписанием и ценами — всё в одном месте.

---

## 🚀 Быстрый старт

```bash
# Установить зависимости 
npm install

# Запустить локальный dev-сервер (Vite)
npm run dev
```

Открыть в браузере: **http://localhost:3000** (или порт, указанный в консоли)

> Приложение использует сборщик **Vite** для быстрой разработки и удобной работы с модулями.

## 📦 Архитектура

### Структура проекта

```
Suluu/
├── index.html          # Точка входа
├── package.json        # Зависимости и скрипты (Vite, Tailwind, ESLint)
├── README.md           # Документация проекта
├── docs/               # Описание модулей
├── css/
│   └── style.css       # Кастомные стили, анимации, компоненты
├── data/
│   └── db.json         # База данных (в формате JSON для внешнего хранения)
└── js/
    ├── app.js          # Инициализация приложения, точка входа
    ├── db.js           # Загрузка данных в память
    ├── state.js        # Глобальное реактивное состояние (state)
    ├── api.js          # Репозиторий данных и эмуляция запросов
    ├── utils.js        # Форматирование, валидация, toast-уведомления
    ├── core/           # Ядро (движок рендеринга и роутер)
    ├── apps/           # Разделенные приложения по ролям (admin, client, master, salon)
    ├── features/       # Фичи приложения (auth, booking, etc)
    ├── components/     # Переиспользуемые UI-компоненты
    └── pages/          # Общие страницы
```

### Ключевые принципы

| Принцип | Реализация |
|---------|------------|
| **State-driven UI** | Единый объект `state`. Любое изменение → `render()` → UI обновлён |
| **DOM-патчинг** | `morphdom` обновляет только изменившиеся DOM-узлы (без мерцания) |
| **Модульность** | Использование ES Modules (import/export) в компонентах и страницах |
| **Сборщик Vite** | Мгновенный запуск и HMR, встроенная поддержка современных JavaScript-модулей |

## 🧩 Функциональные модули

### 👤 Клиент
- Поиск салонов и мастеров с фильтрацией по категориям
- Онлайн-запись: 5-шаговый процесс (Услуга → Мастер → Дата → Время → Подтверждение)
- Выбор «Любой свободный мастер» или конкретного специалиста
- История записей с возможностью редактирования и отмены
- Комментарий к записи

### 🏢 Салон
- Дашборд с ключевыми метриками (записи, доход, мастера)
- Управление записями: подтверждение / отклонение / назначение мастера
- Управление ценами: индивидуальные цены на услуги для каждого салона
- История изменения цен по каждой услуге
- Профиль салона (название, адрес, часы работы)
- Список мастеров салона

### 💇‍♀️ Мастер
- Личное расписание и список клиентов
- Просмотр назначенных записей

### 🛡️ Супер-админ
- Управление справочниками (категории, услуги)
- Обзор всех салонов и мастеров на платформе

## 🔧 Разработка

### Ключевые файлы для изменений

- **Добавить данные** → `data/db.json` / `js/db.js`
- **Изменить логику/UI** → `js/apps/<role>/` или `js/features/<feature>/`
- **Добавить состояние** → `js/state.js`
- **Добавить функцию-утилиту** → `js/utils.js`
- **Новые компоненты** → `js/components/`

### Добавление новой страницы

1. Убедиться в нужных импортах (или создать новую функцию-рендер в `js/apps/<role>/`)
2. В файлах соответствующего приложения (например, `js/apps/client/pages.js`) создать функцию для страницы
3. Добавить навигацию и обработку в нужном роутере или `layout.js`

### Добавление новой роли

1. Добавить пользователя с новой `role` в `js/db.js` (и `data/db.json`)
2. Создать директорию `js/apps/<new-role>`
3. Добавить обработчик рендера в `js/core/router.js` (функция `renderAuthenticatedApp`)

## ⚠️ Известные ограничения

- **Нет персистентности** — данные хранятся in-memory и сбрасываются при полной перезагрузке страницы
- **Tailwind CDN** — на данный момент используется development-версия со скриптом CDN
- **Глобальная привязка DOM** — для обработки событий в `morphdom` часть функций экспортируется глобально или через атрибуты
